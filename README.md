# CS305
Project Reflection

Artemis Financial was the client for this project. The company provides financial services and wanted to improve the security of its public web application so customers could safely transmit sensitive financial information. The primary issue was strengthening the application's security by implementing secure communications using SSL/TLS, verifying data integrity with cryptographic checksums, and identifying software vulnerabilities through dependency analysis.

When identifying the client's software security vulnerabilities, I successfully used dependency-checking tools to locate outdated libraries with known security risks and evaluated the application's overall security posture. I also implemented secure communications by configuring an SSL certificate and adding checksum verification to ensure that transmitted data had not been altered. Coding securely is important because it helps protect sensitive customer information, prevents unauthorized access, and reduces the likelihood of security breaches. Strong software security also increases customer trust, helps organizations meet compliance requirements, and protects a company's reputation and financial stability.

The most challenging part of the vulnerability assessment was configuring the security tools and interpreting the dependency-check results. Some issues required determining whether they represented actual risks or false positives. At the same time, this process was helpful because it demonstrated how automated security tools can quickly identify vulnerabilities that might otherwise be overlooked during manual code reviews.

To increase the application's security, I added multiple layers of protection by enabling HTTPS with SSL/TLS encryption, generating and configuring a self-signed certificate, implementing SHA-256 checksum verification, and reviewing third-party dependencies for known vulnerabilities. In future projects, I would continue using automated vulnerability scanners such as OWASP Dependency-Check, static code analysis tools, secure code reviews, and industry security frameworks like the OWASP Top 10 to evaluate risks and determine the most appropriate mitigation techniques.

I ensured the software remained functional and secure by testing the application after each security enhancement. After refactoring the code, I rebuilt the project, verified that the application executed correctly over HTTPS, confirmed that checksum validation functioned as expected, and reran the dependency analysis to ensure that no new vulnerabilities had been introduced. This testing process helped verify that the security improvements did not negatively affect the application's functionality.

Throughout the project, I used several valuable resources and tools, including Java's Keytool utility to generate SSL certificates, Maven for project management and dependency handling, OWASP Dependency-Check for vulnerability scanning, Spring Boot for application development, and Eclipse as the development environment. I also followed secure coding practices such as validating dependencies, encrypting communications, testing after each modification, and reviewing application behavior after security updates.

If I were presenting this project to a future employer, I would showcase the completed secure web application, the implemented SSL/TLS configuration, checksum verification, dependency-check reports, and the documented vulnerability assessment. These artifacts demonstrate my ability to identify security risks, implement security controls, use industry-standard security tools, and verify that software remains both functional and secure after security enhancements.
